STEP 1 — DATABASE SCHEMA

model User {
  id              String    @id @default(cuid())
  address         String    @unique
  createdAt       DateTime  @default(now())
  tasks           Task[]
  permissions     Permission[]
  spending        Spending[]
}

model Permission {
  id              String    @id @default(cuid())
  userId          String
  user            User      @relation(
                    fields:[userId], 
                    references:[id]
                  )
  tokenAddress    String
  maxAmount       BigInt
  period          Int       // seconds
  expiry          DateTime
  allowedContracts String[]
  active          Boolean   @default(true)
  erc7715Sig      String?
  createdAt       DateTime  @default(now())
  revokedAt       DateTime?
  tasks           Task[]
}

model Task {
  id              String    @id @default(cuid())
  userId          String
  user            User      @relation(
                    fields:[userId],
                    references:[id]
                  )
  permissionId    String
  permission      Permission @relation(
                    fields:[permissionId],
                    references:[id]
                  )
  prompt          String
  status          String    @default("PENDING")
  result          String?
  totalCost       Float     @default(0)
  agentCount      Int       @default(0)
  duration        Int?      // ms
  createdAt       DateTime  @default(now())
  completedAt     DateTime?
  agents          SubAgent[]
  payments        Payment[]
}

model SubAgent {
  id              String    @id @default(cuid())
  taskId          String
  task            Task      @relation(
                    fields:[taskId],
                    references:[id]
                  )
  agentType       String
  budget          Float
  spent           Float     @default(0)
  status          String    @default("PENDING")
  result          String?
  createdAt       DateTime  @default(now())
  completedAt     DateTime?
  payments        Payment[]
}

model Payment {
  id              String    @id @default(cuid())
  taskId          String
  task            Task      @relation(
                    fields:[taskId],
                    references:[id]
                  )
  subAgentId      String?
  subAgent        SubAgent? @relation(
                    fields:[subAgentId],
                    references:[id]
                  )
  amount          Float
  recipient       String
  txHash          String?
  oneShotTx       String?
  status          String    @default("PENDING")
  createdAt       DateTime  @default(now())
}

model Spending {
  id              String    @id @default(cuid())
  userId          String
  user            User      @relation(
                    fields:[userId],
                    references:[id]
                  )
  date            DateTime  @default(now())
  amount          Float
  taskId          String?
}

Functions (complete implementation):
grantPermission(
  address token,
  uint256 maxAmount,
  uint256 period,
  uint256 expiry,
  address[] calldata allowedContracts
) external
— Stores ERC-7715 style permission
— Emits PermissionGranted

revokePermission() external
— User can revoke anytime instantly

spawnSubAgent(
  bytes32 taskId,
  address subAgent,
  address token,
  uint256 budget,
  uint256 expiry
) external onlyOrchestrator
— Creates narrower sub-agent permission
— budget must be < remaining parent permission

executeWithPermission(
  address target,
  bytes calldata data,
  uint256 amount
) external onlySubAgent
— Checks budget not exceeded
— Checks expiry not passed
— Checks target is allowedContract
— Executes via 1Shot relayer
— Updates spent amounts
— Emits PaymentExecuted

completeTask(bytes32 taskId) external onlyOrchestrator
— Marks all sub-agents for task as complete
— Releases any unspent budget

getRemainingBudget(address user) external view
getPermission(address user) external view
isSubAgent(address agent) external view

Events:
PermissionGranted(user, token, maxAmount, expiry)
PermissionRevoked(user, timestamp)
SubAgentSpawned(taskId, subAgent, budget)
PaymentExecuted(subAgent, target, amount, txHash)
TaskCompleted(taskId, totalSpent)

Security:
ReentrancyGuard on executeWithPermission
Pausable emergency stop
Budget checks before every execution
Expiry checks on every call
onlyOrchestrator modifier
onlySubAgent modifier

STEP 5 — API ROUTES

app/api/tasks/create/route.ts:
POST { prompt, permissionId }
Auth: required
1. Validate permission is active + has budget
2. Create task in DB
3. Start orchestrator (background)
4. Return taskId immediately
5. Client connects to WebSocket for updates

app/api/tasks/status/[taskId]/route.ts:
GET → real-time task status
SSE stream of updates

app/api/tasks/history/route.ts:
GET → all completed tasks with costs

app/api/permissions/grant/route.ts:
POST { token, maxAmount, period, expiry,
       allowedContracts }
1. Trigger MetaMask ERC-7715 permission request
2. Store permission in DB + contract
3. Return permission object

app/api/permissions/revoke/route.ts:
POST { permissionId }
1. Call contract.revokePermission()
2. Update DB active = false

app/api/permissions/active/route.ts:
GET → all active permissions for user

app/api/agents/spawn/route.ts:
POST { taskId, agentType, budget }
Internal use by orchestrator

app/api/x402/verify/route.ts:
POST { txHash, expectedAmount, recipient }
DELEGATE's own x402 facilitator endpoint
